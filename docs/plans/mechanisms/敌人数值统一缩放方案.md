# 敌人数值统一缩放方案

> 日期：2026-06-28
> 状态：待审

## 背景

当前问题：

- 子代理从 wuthering.wiki 抓的 Common/Elite 数据是**官方 Lv90 值**（HP 3-4 万 / ATK 6-7 千），但模拟器玩家角色 Lv1 时 ATK 才 400、HP 才 1 万。两个量级差了一个数量级。
- 现有 3 个小怪（幼狼/飞兽/古老幽灵）是模拟器自定的低数值（HP 2200-6500、ATK 260-420），和玩家匹配，但**没有按等级缩放**。
- 世界 BOSS 在副本池里走 `POOL_BOSS_SCALE = 0.10`，把官方 50 万 HP 压到 5 万，但 Common/Elite 没有同等机制。
- 当前 `enemyLevel` 只影响 HP/ATK 的 `level/90` 比例，没区分"世界等级"和"版本膨胀"。

用户需求（5 条）：

1. **等级数值**：我们知道怪物的 90 级数值，通过等级缩放来控制数值。
2. **世界等级**影响地图中出现的怪物的等级（切换世界等级时会刷新副本中怪物的等级），**boss 等级不受影响**，世界等级也应该影响到掉落物品的数量。
3. **世界等级**按照 30%、40%、50% 的数据缩放世界级 BOSS 的数值（这个不需要在前端说出来）。
4. **版本膨胀系数**：随着时间推移、版本前进，世界 BOSS、深塔、海墟、无音区（共同点是可以拿到星声）这些地方的敌人会受到膨胀系数的影响。
5. **世界 BOSS 出现在其他副本时**应该受到缩放（此时应该受到**等级影响 + 缩放系数**两个东西的影响）。

## 设计目标

- **数值有公式可循**：不再拍脑袋定 Common/Elite 的 HP/ATK，所有敌人统一走"Lv90 基准值 × 等级比例 × 世界等级 × 版本膨胀"四因子乘积。
- **量级匹配玩家**：Lv1 副本里小怪和玩家 ATK 400 量级匹配。
- **世界等级有实际作用**：切换索拉Ⅰ/Ⅱ/Ⅲ 不只影响世界 BOSS，也影响副本怪等级和掉落倍率。
- **版本膨胀只作用到星声产出地**：世界 BOSS 讨伐、深塔、海墟、无音区，不影响模拟战训/锻造挑战（这两个是养成料产出，不应随版本通胀）。

## 公式

### 通用敌人最终数值

```
finalHP  = baseHP  × (levelRatio) × worldTierMult × inflationMult × scale
finalATK = baseATK × (levelRatio) × worldTierMult × inflationMult × scale
finalDEF = defForLevel(enemyLevel)   （所有敌人统一 DEF 表）
```

其中：

| 因子 | 来源 | 说明 |
|---|---|---|
| `baseHP` / `baseATK` | `ENEMIES[name].hp/atk` | 官方 Lv90 原始值（世界 BOSS）或模拟器量级基准值（小怪/精英） |
| `levelRatio` | `enemyLevel / 90` | 等级缩放，Lv90 = 1.0 |
| `worldTierMult` | SOL3 等级 | 索拉Ⅰ 0.30 / Ⅱ 0.40 / Ⅲ 0.50 |
| `inflationMult` | 版本号查表 | 见下表 |
| `scale` | 副本配置 `enemyScale` | 组合/单怪微调（如 ×3 杂兵 = 0.5，BOSS = 1.0） |

### 版本膨胀系数表

| 版本段 | inflationMult |
|---|---|
| 1.x | 1.00 |
| 2.x | 1.10 |
| 3.0-3.3 | 1.20 |
| 3.4+ | 1.30 |

> 注：膨胀只作用于"世界 BOSS 讨伐 / 深塔 / 海墟 / 无音区"，**不作用于模拟战训 / 锻造挑战**（养成料产出副本）。

### 小怪/精英的 baseHP/baseATK 校准

参考现有"幼狼"（HP 2500 / ATK 280）和"古老幽灵"（HP 6500 / ATK 420）的模拟器量级，将子代理抓的官方 Lv90 值按下式校准：

```
小怪 baseHP  = 官方 Lv90 HP × 0.10    （3-4 万 → 3-4 千）
小怪 baseATK = 官方 Lv90 ATK × 0.05    （6-7 千 → 300-350）
精英 baseHP  = 官方 Lv90 HP × 0.10    （6-12 万 → 6-12 千）
精英 baseATK = 官方 Lv90 ATK × 0.06    （9-10 千 → 540-600）
```

> 校准系数（0.10 / 0.05 / 0.06）是为了让 Lv90 小怪和模拟器玩家 Lv90 角色（ATK ~1000-1500）量级匹配。
> 这两个系数只在写 `ENEMIES` 数据时用一次，最终存的是校准后的 base 值，不影响运行时公式。

## 影响范围

### 副本场景 × 世界等级 × 等级

| 副本类型 | enemyLevel 来源 | worldTierMult | inflationMult |
|---|---|---|---|
| 模拟战训（40/80 波片） | 玩家世界等级映射 | ✅ | ❌（不膨胀） |
| 锻造挑战（40/80 波片） | 玩家世界等级映射 | ✅ | ❌ |
| 无音区（60 波片） | 玩家世界等级映射 | ✅ | ✅ |
| 世界 BOSS 讨伐 | `S.bossLevels[名]`（独立） | ✅ | ✅ |
| 深塔/海墟 | 关卡配置 | ❌（固定） | ✅ |

### 玩家世界等级 → 副本 enemyLevel 映射

| SOL3 等级 | 副本 enemyLevel |
|---|---|
| 索拉Ⅰ | 40 |
| 索拉Ⅱ | 65 |
| 索拉Ⅲ | 90 |

切换世界等级时，所有副本怪物等级刷新（重渲染即可，无需重启动）。

### 世界 BOSS 在副本池里（如"模拟战训"出现"无妄者"）

走通用公式，但 baseHP 是官方 Lv90 值（50 万级），需要额外 `POOL_BOSS_SCALE = 0.10` 把 base 校准到副本量级：

```
副本池世界 BOSS finalHP = baseHP × POOL_BOSS_SCALE × levelRatio × worldTierMult × scale
```

（注：副本池不叠加 inflationMult，因为模拟战训/锻造挑战不膨胀；如果出现在无音区则叠加。）

## 实施步骤

### Step 1 — 新建 `src/battle/enemyScale.js`

集中所有缩放逻辑：

```js
import { getSol3Level, SOL3_LEVELS } from './dungeon.js';
import { currentVersion } from './dungeon.js';
import { S } from '../state.js';

const POOL_BOSS_SCALE = 0.10;

// 版本膨胀表
const INFLATION_TABLE = [
  { min: '1.0', mult: 1.00 },
  { min: '2.0', mult: 1.10 },
  { min: '3.0', mult: 1.20 },
  { min: '3.4', mult: 1.30 },
];

export function versionInflation(version = currentVersion()) {
  const v = version;
  let mult = 1.0;
  for (const e of INFLATION_TABLE) {
    if (v >= e.min) mult = e.mult;
  }
  return mult;
}

// SOL3 → 副本 enemyLevel 映射
export function sol3ToEnemyLevel(sol3 = getSol3Level()) {
  return { 1: 40, 2: 65, 3: 90 }[sol3] || 40;
}
```

### Step 2 — 改 `src/battle/enemies.js` `spawnEnemy`

接受统一参数对象 `{ enemyLevel, worldTierMult, inflationMult, scale, poolBoss }：

```js
export function spawnEnemy(name, opts = {}) {
  const data = ENEMIES[name];
  if (!data) return null;

  const enemyLevel = opts.enemyLevel || 90;
  const levelRatio = enemyLevel / 90;
  const tierMult = opts.worldTierMult ?? 1.0;
  const inflation = opts.inflationMult ?? 1.0;
  const scale = opts.scale ?? 1.0;
  const poolBoss = opts.poolBoss && (data.class === 'Overlord' || data.class === 'Calamity');

  const baseHP = data.hp * (poolBoss ? POOL_BOSS_SCALE : 1.0);
  const baseATK = data.atk * (poolBoss ? POOL_BOSS_SCALE : 1.0);

  const finalHP = baseHP * levelRatio * tierMult * inflation * scale;
  const finalATK = baseATK * levelRatio * tierMult * inflation * scale;
  const finalDEF = defForLevel(enemyLevel);

  // ... 其余字段不变
}
```

### Step 5 — 校准 14 个新敌人的 baseHP/baseATK

按公式：

```
小怪 baseHP = 官方 Lv90 HP × 0.10
小怪 baseATK = 官方 Lv90 ATK × 0.05
精英 baseHP = 官方 Lv90 HP × 0.10
精英 baseATK = 官方 Lv90 ATK × 0.06
```

例如：
- 剑齿野猪：HP 36000 → 3600，ATK 7000 → 350
- 紫羽鹭：HP 64601 → 6460，ATK 10051 → 603
- 石壁护腕：HP 115809 → 11581，ATK 8979 → 539

校准后 Lv90 时（worldTierMult=0.50, inflation=1.30, scale=1.0）：
- 剑齿野猪 finalHP = 3600 × 1.0 × 0.50 × 1.30 × 1.0 = 2340
- 紫羽鹭 finalHP = 6460 × 1.0 × 0.50 × 1.30 × 1.0 = 4199
- 石壁护腕 finalHP = 11581 × 1.0 × 0.50 × 1.30 × 1.0 = 7528

玩家 Lv1 ATK 400 在 Lv40 副本（enemyLevel=40, tierMult=0.30）：
- 剑齿野猪 finalHP = 3600 × (40/90) × 0.30 × 1.0 × 1.0 = 480
- 玩家普攻 ~400 伤害，约 1-2 击杀

Lv90 副本（enemyLevel=90, tierMult=0.50, 玩家 Lv90 ATK ~1200）：
- 剑齿野猪 finalHP = 3600 × 1.0 × 0.50 × 1.0 × 1.0 = 1800
- 玩家普攻 ~1200 伤害，约 2 击杀

数值合理。
