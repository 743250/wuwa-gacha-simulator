# 副本敌人设计

> 日期：2026-07-01
> 状态：定稿（对齐 src/battle/dungeon.js 当前实装）
> 配套：单体 BOSS 机制见 [../enemies/](../enemies/)，战斗抽象见 [../mechanisms/combat-abstractions.md](../mechanisms/combat-abstractions.md)

## 为什么写这份文档

项目里五个类型的副本——经验本、武器本、无音区、世界BOSS、周本——各自怎么配敌人、为什么这样配，以前散落在代码注释和单 BOSS 文档里，没有一份从玩家视角讲清楚"为什么"的总纲。改副本时无章可循，还出现过把"芙露德莉斯 230 万血"误判成 bug 的事。这份文档把五类副本的设计逻辑、数值依据、版本和世界等级变化一次讲清。

## 设计原则

1. **等级做主线，不靠 scale 压等级**。所有副本走世界等级三档（索拉Ⅰ 30-60 / Ⅱ 60-90 / Ⅲ 90-120），同一个副本在不同世界等级下怪等级不同，血量自然不同。这是玩家心智里的设定——"怪 30 级所以软"。
2. **scale 只在 BOSS 进精英/小怪池时用，且系数克制**。BOSS 的 baseHP 比小怪高一个数量级，同档位同等级下 BOSS 会比小怪硬 8 倍，需要拉平量级时才用 scale。系数统一在 0.3-0.6 之间，不滥用。
3. **养成料本不卡玩家**。经验本和武器本是高频速刷场，所有敌人量级对齐"速刷"定位，BOSS 进池要压到精英量级。
4. **海啸级 BOSS 只进高难本**。无妄者、赫卡忒、鸣式·利维亚坦这种 140 万 baseHP 的，留给无音区/周本，不进经验/武器本。
5. **新手本不出 BOSS**。40 波片档纯小怪+精英，让新手先练手再挑战。
6. **版本膨胀只作用到产星声的副本**——无音区、世界 BOSS、周本。经验本和武器本不膨胀，保证新手不被卡。

## 五类副本各自的设计逻辑

### 模拟战训（经验本）—— 养成料的速刷场

**玩家来这里干嘛**：刷角色升级用的共鸣促剂。每天都要打，最高频的副本。

**配置逻辑**：
- **40 波片新手档**：纯小怪群，不出任何 BOSS。6 个小怪阵容轮换（残象群、焰蜥群、雷风小队等），让玩家不觉得重复，又不会撞上打不过的硬茬。
- **80 波片凝缩档**：精英为主（weight 4），怒涛级 BOSS 稀有遭遇（weight 1，scale 0.35）。BOSS 血量压到精英量级，玩家 2-3 轮能解决，不卡关。
- **不受版本膨胀**：养成料产出，新手不能被卡。1.x 玩家和 3.x 玩家刷经验本遇到的小怪基础值一样。

**为什么不用"训练靶"**：之前新手本塞过聚械机偶当"训练靶"（scale 0.45），但设定违和——新手本不该出世界 BOSS。已删。

**为什么 BOSS scale 是 0.35**：怒涛级 BOSS baseHP 65-104 万，精英 baseHP 11-35 万。同等级下 BOSS 比精英硬 8 倍。scale 0.35 把 BOSS 压到 23-36 万，和精英持平。再低 BOSS 比精英还软，违反直觉；再高就卡关。

### 锻造挑战（武器本）—— 武器线的速刷场

**玩家来这里干嘛**：刷武器升级石。

**配置逻辑**：和经验本完全对称，敌人阵容偏向"构造体/机械/护盾怪"（巡哨机傀、坚岩斗士、异构武装），让武器本有"打硬目标"的体感，和经验本的"打小怪群"区分开。同样不受版本膨胀。

- **40 波片新手档**：小怪+精英混编，不出世界 BOSS。
- **80 波片凝缩档**：精英混编 + 怒涛级 BOSS 影（scale 0.5，比经验本的 0.35 略高，因为武器本定位稍硬）。海啸级 BOSS（赫卡忒、鸣式·利维亚坦）已删，baseHP 140 万压不动。

### 无音区（声骸本）—— 星声的主要产地

**玩家来这里干嘛**：刷星声和声骸养成材料。60 波片一次，星声最重要的稳定来源之一。

**配置逻辑**：因为产星声，必须受版本膨胀——3.x 版本的玩家比 1.x 强，无音区也得跟着变难。每个无音区固定一个守关 BOSS + 固定声骸套装，玩家按需选刷：要熔山裂谷套就打燎照之骑，要凝夜白霜套就打辉萤军势。

**敌人构成**：官方是单波次，守关 BOSS + 1-2 个精英杂兵 + 若干小怪同场出现。模拟器目前简化为只 spawn 守关 BOSS，精英和小怪暂未实装（已知缺口，后续补）。

**版本分层**（按 version 字段）：
- **1.0 基础 9 关**：用怒涛级世界 BOSS 守关（飞廉之猩、燎照之骑、辉萤军势等），Lv90 baseHP 65-100 万。玩家最早接触的无音区，难度入门。
- **2.0 通用 3 关**：罗蕾莱、赫卡忒、叹息古龙，scale 1.6，Lv90 实际血量 152-159 万，给 2.0 玩家新压力。
- **2.0+ 角色专属 9 关**：用海啸级 BOSS 守关（芙露德莉斯、辛吉勒姆等），Lv90 baseHP 100-140 万，对应角色养成需求提升。
- **2.6+ 新增 9 关**：梦魇·赫卡忒、伪作的神王等新 BOSS，Lv90 baseHP 100-140 万。

**和世界 BOSS 的关系**：无音区里的 BOSS 和世界 BOSS 同 ID 同血量——无音区里的飞廉之猩就是世界 BOSS 飞廉之猩，没有"无音区版"和"世界版"的区别。差别只在版本膨胀系数和副本层的 scale。

### 世界 BOSS —— 每日 3 次的硬仗

**玩家来这里干嘛**：刷角色突破材料。每天免费 3 次，不耗波片。

**配置逻辑**：单体硬仗，BOSS 机制要复杂。17 个世界 BOSS 各有专属机制（飞廉之猩抓投只能弹反、燎照之骑叠 10 层爆气、无妄者三阶段五武器切换），是游戏里机制最丰富的 PVE 内容。每日限次意味着玩家不能无限刷，必须认真对待每一场。

**血量定位**：怒涛级，Lv90 baseHP 65-104 万。受版本膨胀（3.4+ ×1.3）。索拉Ⅲ满级 Lv120 时实际血量 108-190 万——这是后期玩家的正常压力。

**讨伐等级**：每个世界 BOSS 有独立讨伐等级，初始 Lv40，打赢 +10，打输 -20，钳位到当前世界等级档位。让玩家有"越打越熟、等级爬升"的进度感。

**已知问题**：dungeon.js 里 worldBoss 条目配的 enemyScale 字段（1.35-1.6）不生效——worldBoss 走 `{worldTier, bossLevel}` 路径，spawnEnemy 里 hpMult=1.0，scale 根本不读。这个字段要么删，要么改代码让 scale 生效，待设计层决定。

### 战歌重奏（周本）—— 每周 3 次的最高难

**玩家来这里干嘛**：刷高阶技能材料。每周共享 3 次限制。

**配置逻辑**：游戏里最强的 PVE 挑战。池子里全是海啸级 BOSS（罗蕾莱、无冠者、赫卡忒），scale 2.5-2.8，索拉Ⅲ Lv120 实际血量 400-460 万——玩家验证养成和操作的终极考场。

**已知问题**：无冠者 baseHP 只有 32 万（剧情 BOSS 级别），即使 scale 2.5 后也才 132 万，远低于同池罗蕾莱 408 万和赫卡忒 458 万。这是基础值本身偏低导致，后续要么提高副本 scale，要么换 BOSS。

## 敌人数值是否对齐官方

**已对齐**：
- 17 个世界 BOSS 的 HP/ATK/DEF：来自 encore.moe API（Lv90 基准），2026-07 已批量同步到 BOSS 单文件文档
- 小怪和精英的 HP/ATK：来自 wuthering.wiki 官方 Lv90 值
- 海啸级 BOSS（芙露德莉斯、辛吉勒姆、赫卡忒、利维亚坦）：来自 encore.moe API

**四档分层清晰**（Lv90 baseHP）：
- 轻波级（小怪）：6-11 万
- 巨浪级（精英）：11.5-35.4 万
- 怒涛级（世界 BOSS）：65.8-104.1 万
- 海啸级（周本 / 角色专属无音区守关）：75-140 万

四档之间没有错配。最硬的精英（巡哨机傀 35.4 万）远低于最弱的世界 BOSS（朔雷之鳞 65.8 万），分层干净。

## 官方异类关卡

官方设计本身就有几个"异类"关卡，守关 BOSS 血量明显低于同档平均值。**这是官方设定，不是模拟器配错**：

- **格洛犸图（Lv90 22 万）**：巨浪级精英，官方让它守绯雪专属无音区（2.0+ 灾厄级档位）。索拉Ⅲ Lv120 实际 36 万，同档其他关卡 170-230 万。
- **无冠者（Lv90 32 万）**：剧情 BOSS，归类怒涛级但血量只有同档 1/3。1.0 无音区·沉日劫光用普通无冠者，梦魇形态才是 100 万。
- **梦魇·哀声鸷（Lv90 77 万）**：菲比专属无音区守关，比同档低 30-45%。
- **海维夏（Lv90 77 万）**：2.6+ 新无音区「逆光跃彩之约」「流金溯真之式」守关，两关都用，比同档低 30-45%。

## 版本变化怎么影响敌人

**核心原则**：版本膨胀只作用到产星声的副本——无音区、世界 BOSS、周本。这三个地方是星声主要来源，玩家随版本变强，敌人也得跟着变强，否则就是无脑碾压。经验本和武器本不膨胀，保证新手不被卡。

**膨胀系数**（按版本号查表）：
- 1.x 版本：×1.0（基准）
- 2.x 版本：×1.1（强 10%）
- 3.0-3.3：×1.2（强 20%）
- 3.4+：×1.3（强 30%）

**玩家体感**：1.x 版本玩家打飞廉之猩可能 80 万血，3.4+ 版本玩家打同一个飞廉之猩就是 104 万血。这是设计意图——玩家角色也在变强，敌人不膨胀就会被一击秒。

**角色专属无音区用海啸级 BOSS**：2.0+ 新增的角色专属无音区（珂莱塔、卡提希娅、布兰特等）用海啸级 BOSS 守关，比 1.0 基础套的怒涛级 BOSS 血量高 50-70%。这是官方设定，对应 2.0+ 角色养成需求提升。

## 世界等级怎么影响敌人

**三档世界等级**对应玩家进度阶段：
- **索拉Ⅰ（世界等级 1）**：怪 30-60 级，新手期，掉落 ×1.0
- **索拉Ⅱ（世界等级 2）**：怪 60-90 级，中期，掉落 ×2.0
- **索拉Ⅲ（世界等级 3）**：怪 90-120 级，后期，掉落 ×3.0

**切换世界等级时**：所有副本等级重置到新档下限。从索拉Ⅱ切到索拉Ⅲ，所有副本怪变成 90 级起步。这让世界等级真正影响"怪的强度"，不只是掉落倍率。

**胜负调整**：每个副本独立记等级。打赢 +10 级，打输 -20 级，钳位到当前档位区间。让玩家有"打不过就降档缓口气、打赢了就稳步爬升"的节奏，不会卡死。

**世界 BOSS 的等级**：世界 BOSS 也有讨伐等级，但同样受世界等级档位钳制。索拉Ⅰ时世界 BOSS 最高 60 级，索拉Ⅲ才能到 120 级。

**量级匹配验证**：索拉Ⅰ Lv30 时，火鬃狼实际血量约 1500，攻击约 290。Lv30 玩家攻击约 400-600，一次技能打 720-1080，2-3 击杀。量级匹配，新手不会被卡。

## 已知缺口和待修

1. **无音区精英 + 小怪混编已实装**：`rollEchoMinions()` 按守关 BOSS 元素从敌人库抽 1-2 个同元素 Elite（scale 0.6）+ 2-3 个同元素小怪（scale 0.5），同 BOSS 同 seed 保证阵容稳定。combat.js 支持 `opts.enemyScales` 数组做 per-enemy scale。官方设定见 `docs/sources/mechanics/tacet-fields-official.md`。
2. **POOL_BOSS_SCALE 死代码已清理**：`enemies.js` 的 `POOL_BOSS_SCALE` 常量和 number 路径的 poolScale 逻辑已删，number 参数直接当 scale 用。
3. **worldBoss 条目的 enemyScale 死字段已删**：17 个 worldBoss 条目的 `enemyScale: 1.35-1.6` 全部删除，worldBoss 走 `{worldTier, bossLevel}` 路径本就不读此字段。
4. **周本无冠者血量偏低**：基础血量 32 万（剧情 BOSS），副本 scale 2.5 后 132 万，远低于同池罗蕾莱 408 万、赫卡忒 458 万。这是官方 baseHP 本身偏低，保持原样不改。
5. **`docs/plans/mechanisms/敌人数值统一缩放方案.md` 已删**：该文档写的 worldTierMult 0.30/0.40/0.50（叠加在 growthRatio 之外）从未在代码实装，代码实际只用 growthRatioTo90 + inflationMult + enemyScale。死代码 `worldTierMult()` 函数也已从 enemies.js 移除。

## 加新副本的检查清单

加一个新副本时，依次想清楚：

1. **它属于哪一类**？是养成料产出（经验/武器本）还是星声产出（无音区/世界BOSS/周本）？这决定了是否受版本膨胀。
2. **敌人怎么选**？养成料本用随机池让玩家不无聊，无音区固定 BOSS 让玩家按需刷，周本用海啸级池子当终极考场。
3. **血量定多少**？小怪 6-11 万，精英 11-35 万，怒涛级 BOSS 65-104 万，海啸级 BOSS 75-140 万。新敌人按官方 Lv90 值入库，不要拍脑袋。
4. **BOSS 进池要不要 scale**？BOSS 进精英/小怪池时，scale 0.3-0.6 拉到档位量级。不要用 scale 压等级——等级由世界等级三档控制。
5. **开启等级**？低级本给新手，高级本给中后期，别让 Lv1 玩家撞上 200 万血 BOSS。
6. **是否引入新 BOSS**？如果是，先写 [../enemies/<BOSS名>.md](../enemies/) 设计文档，讲清机制和数值依据，再动代码。
