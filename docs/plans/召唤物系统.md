# 召唤物系统 · 通用机制设计

> 状态: 📝 草稿(2026-06-30 新增)
> 起因: 弗洛洛赫卡忒需要独立 HP + 挡刀 + 不可控,现有"召唤物"(折枝墨鹤/莫特斐浮翼狂想)都是计数追击的伪召唤,表达不了。

## 为什么需要通用机制

鸣潮里"召唤物"是一类角色设计模式:赫卡忒(弗洛洛)、岁主系(今汐/岁主相关)、可能还有 3.x 后续角色。如果每个角色各自实现一套,combat.js 会被 ad-hoc 代码淹没。抽成通用接口,新角色只需提供数据 + 钩子。

## 与现有"伪召唤"的区别

| 维度 | 伪召唤(折枝墨鹤) | 真召唤物(赫卡忒) |
|---|---|---|
| HP | 无,只是计数 | 独立 HP 实体 |
| 受伤 | 不受伤 | 可被攻击,可死亡 |
| 行动 | 主人攻击时追击 | 独立行动(每回合自动攻击) |
| 控制 | 无 | 玩家不可控制、不可切换 |
| 持续 | 计数耗尽或领域结束 | HP 归零或 duration 归零 |

两套机制并存,不互相替代。折枝墨鹤继续用 `summonField` 计数,赫卡忒走新系统。

## 数据结构

召唤物挂在 `battle.summons` 数组上,**不进 `battle.team`**(玩家切不到)。

```js
battle.summons = [{
  id: 'frolo_hecate_0',         // 唯一 id(同主人多次召唤用 idx 区分)
  name: '赫卡忒',
  ownerIdx: 0,                   // 主人在 team 中的索引
  ownerName: '弗洛洛',
  hp: 10775, hpMax: 10775,
  atk: 438, def: 1137,           // 继承主人属性快照(召唤时锁定)
  element: '湮灭',
  duration: 3,                    // 剩余回合数
  alive: true,
  // 钩子(由角色机制文件提供)
  onTurnStart: (summon, battle) => {},    // 每回合开始时自动行动
  onOwnerDamaged: (summon, incomingDmg, battle) => dmg,  // 主人受伤前拦截,返回实际打主人的伤害
  onDeath: (summon, battle) => {},        // 死亡时回调
}]
```

## 伤害路由(dealDamage 改造)

`combat.js:296` 的 `dealDamage(target, dmg)` 在进入主体逻辑前,先检查 target 是否有存活的挡刀召唤物:

```js
function dealDamage(target, dmg) {
  // ★ 召唤物挡刀:target 是玩家角色,且有存活召唤物时,优先打召唤物
  if (target._isPlayerUnit && !target._isSummon) {
    const ownerIdx = battle.team.indexOf(target);
    const shielder = battle.summons?.find(s => s.alive && s.ownerIdx === ownerIdx && s.onOwnerDamaged);
    if (shielder) {
      const remaining = shielder.onOwnerDamaged(shielder, dmg, battle);
      // remaining 是 overflow 打主人的伤害
      if (remaining <= 0) return 0;  // 全被召唤物吃掉
      dmg = remaining;
    }
  }
  // ... 原有 defense buff / 护盾 / 主体扣血逻辑
}
```

召唤物自身受伤走简化路径(不触发主人的 defense buff / 护盾):

```js
function damageSummon(summon, dmg) {
  summon.hp -= dmg;
  if (summon.hp <= 0) {
    summon.hp = 0;
    summon.alive = false;
    summon.onDeath?.(summon, _currentBattle);
  }
}
```

## 回合循环

召唤物行动在**主人回合开始时**自动触发,不消耗主人 AP:

```js
// doAction 内,玩家行动前
function tickSummons(battle, ownerIdx) {
  for (const s of battle.summons) {
    if (!s.alive || s.ownerIdx !== ownerIdx) continue;
    s.onTurnStart?.(s, battle);
  }
}
```

回合结束时 `duration -1`,归零则消失。

## 切人交互

召唤物**不进 `battle.team`**,所以:
- `battle.team` 数组只有玩家角色,切换逻辑天然不受影响
- 切人时主人退出战场,召唤物随之消失(由主人机制文件的 `switchOut` 钩子清理)
- 召唤物不能被玩家选中攻击(敌人 AI 也不会主动打召唤物,除非有特殊机制)

## 战斗 UI 适配

`renderTeam()` 在每个角色卡片下方,如果有存活召唤物,渲染一个小的 HP 条卡片:

```
┌─ 弗洛洛 (active) ────┐
│ HP ████████░░ 8020/10775 │
│ 能量 ██░░░░░░░░ 24/100   │
│ 余响 ██████░░░░ 14/24    │
│ 乐声 ●●●●○○ 3/6          │
├─────────────────────────┤
│ 🢀 赫卡忒 (召唤物)        │
│ HP ██████░░░░ 6430/10775 │
│ 持续 2 回合              │
└─────────────────────────┘
```

召唤物卡片:
- 不可点击(无切换)
- 显示 HP / 持续回合
- 用不同颜色或边框区分(如紫色边框)标识"召唤物"
- 死亡时显示"已消散"短暂提示后移除

## 开放问题

1. **多个召唤物共存**:同一角色能否同时有多个召唤物?当前设计 `battle.summons` 是数组,支持。但 UI 渲染需要考虑多召唤物的布局。
2. **召唤物受 AO E**:敌人 AOE 打全队时,召唤物是否也受伤?当前设计召唤物只在主人被指定攻击时挡刀,AOE 不挡。后续可扩展。
3. **召唤物攻击目标选择**:自动攻击默认打主目标,是否需要"仇恨"机制?当前简化为主目标。
4. **召唤物属性快照时机**:召唤时锁定(主人 buff 不影响召唤物) vs 实时同步(主人 buff 加成召唤物)。当前设计快照锁定,简化计算。
