### 1.2 世界等级缩放

SOL3 世界等级决定取 Lv90 基准的百分比：

| 世界等级 | 倍率 | 说明 |
|---|---|---|
| 索拉Ⅰ | ×0.60 | 新手期，HP≈280k-350k |
| 索拉Ⅱ | ×0.70 | 中期，HP≈320k-410k |
| 索拉Ⅲ | ×0.80 | 后期，HP≈370k-470k |

> 不再使用旧的 `SOL3_LEVELS` 固定 scale（1.0/1.3/1.6），改用官方 Lv90 基准 × 世界倍率。

### 1.3 BOSS 个人等级系统

每个世界 BOSS 有独立的「讨伐等级」，记录被击败和失败次数：

- **初始等级**：Lv40
- **每次被击败**：+10 级（上限 Lv90）
- **每次挑战失败**（全队阵亡）：-20 级（下限 Lv40）
- 等级影响：在 Lv90 基准上，按等级比例缩放 HP/ATK（DEF 按通用 DEF 表）

```
实际 HP  = Lv90基准HP × 世界倍率 × (当前等级/90)
实际 ATK = Lv90基准ATK × 世界倍率 × (当前等级/90)
实际 DEF = DEF表[当前等级]（通用公式 800 + (lv/10)×80）
```

Lv40 时约为 Lv90 的 44%，Lv90 时即为世界倍率满值。

### 1.4 副本入口调整

世界 BOSS 副本不再使用 `enemyScale` 固定倍率，改用世界等级 + BOSS 个人等级组合。`spawnEnemy` 需要支持接收 HP/ATK/DEF 绝对数值（而非单纯的 scale 参数）。

## 二、17 BOSS 机制移植设计

> 每个 BOSS 的设计格式：**官方核心 → 模拟器抽象 → 新 mechanic type**

### 01 燎照之骑 Inferno Rider ⭐⭐⭐

**官方核心**：Fusion Burst 叠层（10 层满爆）+ 双阶段（骑摩托 → 步行 @75%HP）

**模拟器抽象**：

```
灼伤标记（Fusion Burst 折合）：
  - 被热熔攻击命中 +N 层灼伤（叠层本身不扣血）
  - 每层灼伤：回合结束时扣血 = 攻击×80%（dot）
  - 上限 5 层
  - 叠满 5 层：立即清空层数，额外爆炸 = 攻击×300%

双阶段：
  阶段一（HP ≥ 75% · 骑摩托）：
    每次攻击附带 2 层灼伤
  阶段二（HP < 75% · 步行）：
    每次攻击给全体角色各 1 层灼伤（群体叠层压力）
```

**mechanic type**：`inferno_marks`（新）

**参数**：
```js
mechanic: {
  type: 'inferno_marks',
  markDmgPct: 0.80,    // 每层dot = ATK × 80%
  maxStacks: 5,         // 5 层满爆
  burstMult: 3.0,       // 满层爆炸 = ATK × 300%
  phase1Stacks: 2,      // P1 每次攻击给 2 层
  phase2Stacks: 1,      // P2 每次攻击给全队 1 层
  threshold: 0.75       // 阶段分界线
}
```

### 02 飞廉之猩 Feilian Beringal ⭐⭐

**官方核心**：抓投（只能弹反，不可闪避）+ 冲击波 AOE + 近战连段

**模拟器抽象**：

```
抓投（核心机制）：
  - 每 4 回合，对当前角色释放抓投
  - 伤害 = ATK × 250%（极高）
  - 不可闪避（无视 dodge），不可格挡
  - 只能弹反：若本回合角色使用了"重击"→ 弹反成功，抓投 miss

地面冲击波：
  - 每 3 回合，全队 AOE = ATK × 80%

低血量狂暴：
  - HP < 40%：ATK +40%，抓投频率提升到每 2 回合
```

**mechanic type**：`baringal_grab`（新）

**官方核心**：双形态（锤 ←→ 翼）+ Overclock 红温无敌 + 地面延迟爆破

**模拟器抽象**：

```

  - 每 3 回合切换一次形态

  - 翼形态：飞空弹幕，3 发随机目标（每发 ATK × 80%）

Overclock 红温（HP < 30%）：
  - 首次触发：BOSS 进入 1 回合无敌 + 攻击欲望翻倍
  - 无敌结束后，每回合攻击次数 +1（持续 3 回合）
```

## 四、兼容性与过渡

### 4.1 现有的副本入口

当前 `dungeon.js` 中世界 BOSS 副本（`type: 'worldBoss'`）的 `enemyScale` 将被替换为世界等级系统。副本掉落不变（已有 tier 分级）。

### 4.2 旧 mechanic 清理

这 17 个 BOSS 当前在 `enemies.js` 中使用的是简化 mechanic（burn_team / enrage / shield / dive / thunder_chain / aoe_freeze / data_lock / minion / reflect），移植后将全部替换为新 mechanic type。

旧 mechanic type（如 `burn_team`、`dive`、`thunder_chain` 等）保留在 `enemyMechanics.js` 中，因为深塔/副本的 encounterPool 可能仍引用它们；未来可逐步迁移。

### 4.3 BOSS 等级持久化

BOSS 讨伐等级需存入 `S`（全局状态）中，格式：
```js
S.bossLevels = {
  '燎照之骑': 50,
  '飞廉之猩': 40,
  // ...
}
```

副本面板需显示当前讨伐等级。

### 4.4 战斗 UI 适配

部分新机制需要在战斗日志 / UI 中体现：
- 灼伤标记层数显示（debuff 图标）
- 湮灭之蚀层数显示
- 冰翼盾／护盾条显示（已有 shield 字段）
- 飞空无敌状态提示
- 浮空剑存活指示
- 残骸可投掷提示

## 五、实施顺序

按复杂度从低到高、Boss 在副本中的出现频率排序：

| 批次 | BOSS | 复杂度 | 原因 |
|---|---|---|---|
| **第 1 批** | 梦魇亚当、聚械机偶、朔雷之鳞 | 低 | 机制简单，快速完成积累经验 |
| **第 2 批** | 燎照之骑、飞廉之猩、云闪之鳞、哀声鸷 | 中 | 核心 BOSS，出现频率高 |
| **第 3 批** | 无常凶鹭、辉萤军势、鸣钟之龟、荣耀狮像 | 中 | 特色机制，中等复杂度 |
| **第 4 批** | 异构武装、无归的谬误、叹息古龙 | 高 | 多阶段/多技能池 |
| **第 5 批** | 罗蕾莱、海之女、无妄者 | 高-极高 | 最复杂的 3 个 |
