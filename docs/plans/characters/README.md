# 角色设计指导

## 模拟器定位

本模拟器叫**鸣潮抽卡模拟器**。主题是抽卡,战斗是附带的——战斗模拟只是为了让抽出来的强力角色和武器有相应体验,不是要把鸣潮的战斗系统完整复刻。

因此每个角色的移植力度不同。

## 移植力度

移植力度按 [../../sources/tier-list.md](../../sources/tier-list.md) 强度榜分档,**越强力机制保留越完整**:

- 强力角色(原版游戏设计的氪金点)机制要尽量完整保留——这是让玩家感受到"抽到强角色"那种压制感的核心。
- 弱力角色机制可简化、数值可保守,**机制照样有趣,数值要克制**。
- SS/S 级角色由我和用户一同讨论完成;A/B/C 级用工厂版 `customLines` + `CHAIN_BATTLE_EFFECTS` 标准 effect,不写专属 helper。
- 每次设计新角色前**先看 tier-list**,不要再凭印象拍脑袋。

## 基本思路

1. **基本数据**:攻击力 / 防御力 / 血量等直接套官方数值(已在 [../../sources/characters/](../../sources/characters/) 保存)。
2. **技能与共鸣链**:容易移植到模拟器的就保留;涉及新机制的要看移植力度——力度高才加新机制,力度低就折算到现有 effect。
3. **移植力度和定位**:按强度榜决定。SS 拉满,S 保留主体,A/B/C 保守。

## 移植级别说明

- **S 级**(守岸人 / 忌炎 / 吟霖 / 卡提希娅):完整结构化战斗,包含 chains.js 专属 effect + combat.js 专属 helper + 5 文件全改,有独有状态机。
- **A 级 → 工厂**:用 [render.js `makeSkillLines` 工厂](../../src/ui/render.js) 生成 customLines(公式 tooltip + 配置驱动),战斗折算靠 [chains.js `CHAIN_BATTLE_EFFECTS`](../../src/battle/chains.js) 标准 effect。机制差异在 chain effect 数值 + 工厂配置,不写专属 combat helper。

## 战斗系数基线(2026-06-24 重校)

> 旧版"解放 300% atk / 3 AP"实际收益是 100%/AP,跟 3 普攻打平,加上要攒 125 能量根本不值得放——战略上反向。改为:

| 行动 | AP | 倍率 | 输出 / AP(单体)|
|---|---|---|---|
| 普攻 | 1 | 100% | 100% |
| 共鸣技能 | 1(CD3)| 180% | 180%(高但受 CD 限制)|
| 重击 | 2 | 220% | 110% |
| **共鸣解放** | **3** | **主目标 400% / 副目标 200%** | **主 133%/AP**,3 敌 AOE 总 800% |

实装位置:[combat.js:doBurst](../../src/battle/combat.js),主目标 = `battle.targetIdx`(UI 选中的那只)。
