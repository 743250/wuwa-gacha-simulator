# 决策日志（ADR）

记录项目中**非显而易见**的决策——为什么这么定、为什么不那么定。代码能看出来的"是什么"不记，只记"为什么"。

## 何时记一条

- 用户口头拍板了一个机制/数值/边界（没进设计文档，但后续会有人想改）
- 多个合理方案里选了一个，另一个的代价值得留痕
- 老存档/老行为被改了，需要解释"为什么以前是 A 现在是 B"
- **不记**：纯 bug 修复、纯重构、纯文案改动

## 格式

```
## YYYY-MM-DD · 标题

**决策**：一句话结论。
**背景**：为什么需要决定（什么冲突、什么触发）。
**取舍**：考虑过的另一方案 + 为什么没选。
**影响**：动了哪些文件 / 老存档怎么办 / 后续不能随便改的地方。
```

文件命名：`docs/decisions/NNNN-简短标题.md`，NNNN 为顺序号。
