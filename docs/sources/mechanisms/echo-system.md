# 声骸系统（Echo System）官方数据

采集日期：2026-06-25 · 来源：库街区官方共建 Wiki / 游戏内数据

## 一、系统概述

声骸（Echo）是鸣潮的核心养成系统，相当于传统 RPG 的"装备 + 圣遗物 + 召唤兽"三合一。

| 维度 | 说明 |
|------|------|
| 定位 | 每个角色最多佩戴 5 个声骸 |
| COST 上限 | 角色 COST 上限 = 数据坞等级决定（满级 12），常用 10~12 |
| 类型 | 分 COST 1 / 3 / 4 三档（早期有 COST 2 后来合并入 COST 3） |
| 成长 | 升级 + 调谐（解锁副词条） |
| 套装 | 2 件套 / 5 件套效果（可跨位面组合） |
| 技能 | 主声骸位（第 1 个）可释放声骸技能 |

### 核心养成闭环

```
刷声骸（无音区 / 大世界探索）
  → 筛选主词条正确的声骸
    → 升级 + 调谐（赌副词条）
      → 凑套装效果
        → 强化角色面板 × 战斗能力
```

## 二、数据坞（Data Bank / 数据坞）

数据坞是声骸系统的核心进度系统。

| 等级 | 所需经验 | 解锁效果 |
|------|---------|---------|
| 1 | 0 | 基础声骸掉率，COST 上限 4 |
| 2 | 100 | COST 上限 5 |
| 3 | 200 | COST 上限 6 |
| 4 | 400 | COST 上限 7 |
| 5 | 800 | 解锁金色（5★）声骸概率，COST 上限 8 |
| 6 | 1200 | COST 上限 9 |
| 7 | 1600 | COST 上限 10 |
| 8 | 2000 | COST 上限 11，金色声骸 100% |
| 9 | 2400 | COST 上限 12，吸收概率提升 |
| 10 | — | 满级，声骸必掉 |

> 数据坞经验通过首次吸收/升级声骸获得。每种声骸首次吸收 +20 经验。

**吸收机制**：击败敌人后有概率掉落声骸。数据坞等级越高，掉落概率越高。
- Lv.1~4：仅掉落紫（4★），概率 < 20%
- Lv.5~7：有概率掉金（5★），随等级提升
- Lv.8+：必掉金（5★）
- Lv.10：吸收概率 100%（不再有吸收失败）

## 三、声骸 COST 系统

声骸按 COST 分为三档：

| COST | 类型 | 主词条池 | 说明 |
|------|------|---------|------|
| 1 | 普通声骸（Common）| 固定百分比/数值 | 小怪掉落，低成本凑套装 |
| 3 | 精英声骸（Elite）| 多样化 | 精英怪掉落，核心属性件 |
| 4 | BOSS声骸（Overlord/Calamity）| 高数值 | 世界BOSS / 周本BOSS掉落 |

**COST 分配示例（上限 12）**：
- 43311：4 + 3 + 3 + 1 + 1 = 12（主流输出配置）
- 44111：4 + 4 + 1 + 1 + 1 = 12（特殊配置）
- 33333：3 + 3 + 3 + 3 + 3 = 12

> 第 1 个声骸位为主声骸位（显示在角色背后），释放声骸技能。主声骸位可切换为任意已吸收的声骸外观。

### 6.2 调谐系统

声骸升级时解锁调谐词条：

| 声骸等级 | 解锁副词条数 | 所需经验 |
|-----------|-------------|--------|
| 0→4 | 0 | 少量 |
| 4→9 | 1 | 中量 |
| 9→14 | 2 | 较多 |
| 14→19 | 3 | 大量 |
| 19→25 | 4（满） | 最大 |
| 5★ Lv.25 | 共 5 条 | 总计约 540,000 经验 |

- 每级调谐从词条池中随机抽取（不重复已有词条）
- 词条值在档位范围内浮动
- 可使用【调谐器】重roll词条

## 十二、声骸在战斗中的角色

### 对模拟器的参考意义

声骸系统为本模拟器提供了以下设计参考：

1. **面板构成**：声骸提供了角色 50%~70% 的战斗面板（攻击/暴击/暴伤/属性伤害）
2. **增伤区间**：声骸套装效果覆盖了增伤/属性伤/全伤等多个乘区
3. **战斗节奏**：声骸技能可以看作免费的额外技能，影响 AP 策略
4. **养成目标**：声骸调谐（赌词条）是终局玩法的核心消耗

### 模拟器暂未实装的部分

- 声骸的套装系统（2/5 件效果）
- 声骸的主/副词条随机生成
- 声骸技能（主动释放）
- 声骸升级与调谐
- 数据坞等级进度
- 无音区刷取

> 数值（如词条浮动范围、声骸技能倍率）以实际版本为准，此处记录通用框架。
