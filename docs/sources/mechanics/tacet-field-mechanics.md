# 鸣潮 无音区（Tacet Field）官方机制归档

> 国服 3.4 版本为准。归档日期：2026-07-01
> 研究方式：encore.moe API（怪物列表/详细数据） + 模拟器代码对比
> ⚠️ 社区文章（bilibili wiki / 库街区 / Fandom）均因反爬/空页/Cloudflare 无法直接抓取，以下机制数据来自 encore.moe 怪物数据和模拟器现有配置推断

---

## 1. 核心机制

### 1.1 什么是无音区
无音区（Tacet Field / 无音区·残响）是鸣潮中的声骸养成副本，消耗体力（结晶波片）挑战敌人，获取声骸及其相关素材。

### 1.2 体力消耗
- **60 波片/次**（对应模拟器 `cost: 60`）
- 可使用凝缩波片双倍消耗（80 波片档 → 双倍产出，但官方无音区固定 60 波片，凝缩后为 120 波片？模拟器未实现凝缩波片）

### 1.3 产出
- **声骸**：主要产出，对应关卡绑定的声骸套装
- **共鸣促剂/经验**：副产物
- **武器突破石**：副产物
- **星声**：少量（模拟器自定义）
- **贝币**：少量

> 注：模拟器将声骸调谐器（密音筒/调谐器）等子系统折算为 exp + 武器石，见 dungeon.js 第 10-14 行注释。

### 1.4 解锁等级
| 难度档位 | 推荐等级 | 解锁条件 | 官方对应 |
|---------|---------|---------|---------|
| Lv.50 | 30+ | 完成主线/达到 SOL3 Phase 2 | 无音区·常规 |
| Lv.70 | 50+ | SOL3 Phase 3+ | 无音区·高阶 |
| Lv.80 | 60+ | 2.0+ 版本内容 | 角色专属声骸副本 |

> 官方具体解锁等级：未从社区文章确认。上述为模拟器现有配置推断。

### 1.5 难度档位
官方 Tacet Field 根据玩家 SOL3 Phase（索拉世界等级）动态调整敌人等级。SOL3 Phase 1-8 对应不同的敌人等级和掉落质量。

模拟器简化为 3 档 SOL3 等级（见 `dungeon.js` SOL3_LEVELS）：
- 索拉 I：×0.30
- 索拉 II：×0.40
- 索拉 III：×0.50

### 1.6 轮换机制
- 无音区**没有每日/每周轮换**。每个声骸套装有固定的 Tacet Field 关卡。
- 新版本增加新声骸套装时，同步新增对应的 Tacet Field 关卡。
- 模拟器的 `encounterPool` 轮换机制（每日换敌人组合）是**模拟器自创**，非官方机制。

### 1.7 声骸 COST 与套装绑定
- 每个 Tacet Field 关卡固定掉落特定声骸套装（如「凌冽决断之心」、「流云逝尽之空」等）
- 声骸 COST（1/3/4）不直接与关卡绑定，而是在声骸本身属性中定义
- 模拟器的 `echo_set` 字段实现此绑定

### 3.3 敌人数值公式（encore.moe API 确认）

每个怪物有基准属性 `Properties`（Lv1 基础值）和 `GrowthRates`（每级增长率）：

- **HP at level N** = `Properties.LifeMax.Value × GrowthRates[N].LifeMaxRatio / 10000`
- **ATK at level N** = `Properties.Atk.Value × GrowthRates[N].AtkRatio / 10000`
- **DEF** = 80 + (N × 80)（通用公式，部分怪物有额外防御系数）

示例（Lv90 对比）：

| 怪物 | 稀有度 | Lv1 HP | Lv1 ATK | Lv90 HP | Lv90 ATK |
|------|-------|--------|---------|---------|---------|
| 先锋幼岩 | 轻波级 | 129 | 102 | 60,300 | 3,417 |
| 紫羽鹭 | 巨浪级 | 368 | 150 | 172,017 | 5,026 |
| 燎照之骑 | 怒涛级 | 1,784 | 133 | 833,910 | 4,456 |

注意：怒涛级 BOSS 的基础 ATK 不一定比精英高，但 HP 增长曲线更陡（LifeMaxRatio 增长更快）。

### 4.5 敌人缩放系数

当前模拟器对非世界 BOSS 的缩放公式为：

```js
// 副本敌人按等级比例缩放（Lv90 为基准）
if (!isWorldBoss) {
  const lvRatio = opts.enemyLevel / 90;
  finalHp = data.hp * hpMult * lvRatio;
  finalAtk = data.atk * atkMult * lvRatio;
}
```

这与官方 GrowthRates 的线性缩放不同（官方有独立的每级增长率，不是简单的 lv/90 比例）。如果要校准，应该直接用 encore.moe API 的 GrowthRates 数据。

## 6. 待确认

以下是搜不到/有争议的点，需用户决定：

1. **官方 Tacet Field 敌人组合**：每个无音区关卡具体配置哪些敌人（杂兵+精英+BOSS 混编，还是仅 BOSS），社区文章无法抓取，需要手动浏览确认。

2. **encore.moe 小怪/精英的 Lv90 官方数值**：API 确实暴露了 GrowthRates 数据，可以通过 `GrowthRates[90]` 计算每个等级的官方数值。当前 enemies.js 中的小怪/精英数据可据此校准。

3. **角色专属副本的 BOSS 选择**：模拟器假设每个角色专属声骸副本对应一个 BOSS（如流云逝尽之空→飞廉之猩），需确认官方是否如此配置。

4. **Lv50 无音区是否有多 BOSS 混编**：模拟器 silent_1 使用 encounterPool 每天轮换敌人组合，这与官方 Tacet Field 的"固定敌人+固定套装"设计不符。建议改为：每个无音区关卡固定敌人（如仅一个 BOSS 或固定组合），移除轮换池。

5. **声骸 COST 数据**：encore.moe echo 数据中 `Cost` 字段缺失（266 个声骸均为 undefined），声骸 COST（1/3/4）需从其他来源获取。

6. **模拟器自定义产出（星声）**：无音区产出星声（astrite: 10）是模拟器自定义。官方无音区是否产出星声需确认。

7. **凝缩波片双倍档**：模拟器未实现无音区的 120 波片双倍档（仅在模拟战训/锻造挑战有 80 波片凝缩档）。

8. **新声骸的 FetterGroup 到模拟器 echo_set 的映射**：encore.moe 返回的 FetterGroup 名（如「流云逝尽之空」「凌冽决断之心」）与模拟器使用的 echo_set key（如 `cartethyia_wind`、`carlotta_skill`）之间的映射关系需要手动建立。
