## 1. 声骸调谐词条机制

### 1.1 锁定与副词条激活（已确认）

源: `/src/equip/echoActions.js` 第 51-56 行、第 195-198 行

- 金色声骸共 5 个副词条槽位
- 初始 0 条解锁，每升 5 级(5/10/15/20/25)激活下一个未解锁槽位
- 解锁时随机从副词条池中选一条(不重复)
- 现有实装: `levelUpEcho` 中 `if (echo.level % 5 === 0)` 激活 `nextLocked`

### 1.2 调谐器作用（已确认，与官方一致）

源: `/src/equip/echoActions.js` 第 262-277 行 `retuneEchoSubStat`

**调谐器只改副词条数值，不改词条类型。**
- 消耗 1 调谐器，指定一个已解锁副词条
- 词条类型不变(保留 key)
- 数值在 SUB_STAT_POOL 定义的范围(min~max)内完全重新随机
- 玩家不能选目标词条
- 也不能锁定/保留某些词条不重 roll(整个值重随)

### 1.3 候选池（已确认）

源: `/src/data/echoes.js` 第 434-448 行 `SUB_STAT_POOL`

- 候选池 = 13 种词条: 暴击率/暴击伤害/攻击%/生命%/防御%/攻击固定/生命固定/防御固定/共鸣效率/普攻伤害/技能伤害/解放伤害/重击伤害
- 不按主词条/套装/元素限定 — 所有类型共享同一池
- 池子不重复(同一副词条不会出现 2 次)
- 5 条副词条从 13 种中选 5 种

### 1.4 官方调谐机制的社区确认

由于 wiki 被反爬，调谐机制来自中文玩家社区共识：
- 调谐器(日文:チューナー / 英文:Tuner)只能重roll副词条数值(数值幅度随机)
- 不能改变词条类型 → 金词条初始解锁的 type 就是最终 type
- 没有"锁定后重投保留锁定项"机制(这是隔壁游戏的 feature，鸣潮目前没有)
- 每次消耗 1 个调谐器
- 调谐操作可无限次对同一词条进行

### 1.5 未找到确切官方来源

- 鸣潮官方没有公开的"调谐机制详解"页面(未找到类似蓝贴或公告)
- 当前实装与玩家社区教程(如 NGA/米游社/B站攻略)一致
- 如有官方改动以游戏内实装为准

### 3.3 锚点已知(满级值),按社区公式推导

值(Lv) = 值(Lv0) + (值(Lv25) - 值(Lv0)) × (Lv / 25) × curve_factor(segment)

### 3.5 需要注意

- 副词条升级(解锁新槽位)时,已解锁槽位的值**不会变化**
- 副副词条在解锁时按 min~max 均匀随机取一个值
- 只有调谐器可以重 roll 已解锁的副词条值
- **不存在"强化的随机增长"**(不像隔壁原神/星铁的那种升级随机涨副词条)
- 鸣潮的副词条在解锁那一刻值就定死,后续只靠调谐器改

## 已知不足与待确认

1. 主词条 Lv.0 精确值: 需要游戏内截图实测或数据挖掘
2. 各等级(5/10/15/20/25)精确主词条数值: 需要实测
3. 调谐器商店精确限量: 版本更新频繁,需游戏内确认
4. 2.0 版本后声骸 25 级上限对主词条的影响: 当前社区数据可能已更新
5. 副词条升级数值: 现有代码已实现"解锁时定值",但如官方有"解锁后值随等级微调"需确认

## 对模拟器实现的建议

2. **主词条补充**: 补充 Lv.0 的占位值(按满级 40% 推算),后续有实测数据再校准
3. **副词条 OK**: 当前实现与官方一致
4. **调谐器获取**: 当前声骸分解返还已实现;商店兑换尚未实现(如需要)
