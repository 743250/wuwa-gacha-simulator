# 角色设计指南

本文档定义“从官方角色到模拟器角色”的设计流程。具体每个角色的设计稿写在 [../character-design.md](../character-design.md)，官方原文资料写在 [../official/characters/](../official/characters/)。

## 资料顺序

1. 先看官方技能组：资源怎么攒、形态怎么进入、持续多久、怎么退出、哪些按钮被替换。
2. 再看官方 6 条共鸣链：标题、数值、强化对象。
3. 再看强度榜：[../tier-list.md](../tier-list.md)，用于决定数值天花板。
4. 最后写模拟器抽象：AP 回合制保留哪个核心决策。

## 必须区分的三件事

| 类型 | 说明 | 写在哪里 |
|---|---|---|
| 官方原文 | 官方技能、共鸣链、武器、敌人机制 | [../official/](../official/) |
| 模拟器抽象 | 把 ACT 机制折成 AP 回合制后的设计 | [../character-design.md](../character-design.md) |
| 实装指南 | 代码路径、tooltip 标准、移植步骤 | [../../CLAUDE.md](../../CLAUDE.md) |

## 改角色机制前的规则

- 不因为 CLAUDE.md 或旧文档一句话就直接改角色数值。
- 如果官方资料、当前实装、技能文案三者不一致：
  1. 先记录差异。
  2. 查官方资料。
  3. 问用户采用“官方贴近”还是“模拟器自定义折算”。
- 架构优化不应顺手改角色强度。

## 设计输出格式

每个角色建议包含：

```markdown
## 版本 · 角色名

- 官方资料：[official/characters/xxx.md](../official/characters/xxx.md)
- 强度 Tier：SS / S / A / B / C
- 移植级别：S / A / B / C

### 官方核心

### 模拟器保留

### 模拟器舍弃 / 合并

### 6 链折算

### 数值与 tooltip 规则

### 待确认问题
```

## 守岸人示例提醒

守岸人官方 1 链是“范围 + 持续 + 洞悉后不结束星域”。如果模拟器使用“1 链星域增益 ×2.5”，必须标注为模拟器自定义强度折算，而不能写成官方效果。
